multi cloud et hybrid cloud différence 
Multi-cloud

    Définition : Utilisation simultanée de plusieurs services cloud publics provenant de différents fournisseurs (ex : AWS + Azure + Google Cloud).

    But :

        Éviter la dépendance à un seul fournisseur (lock-in).

        Choisir le meilleur service pour chaque besoin spécifique.

        Optimiser coûts, performances, et résilience.

    Exemple : Une entreprise utilise AWS pour ses machines virtuelles, Azure pour ses bases de données, et Google Cloud pour l’analyse de données.


Hybrid cloud (Cloud hybride)

    Définition : Combinaison intégrée d’un cloud privé (sur site ou dédié) et d’un ou plusieurs clouds publics, avec orchestration et gestion unifiées.

    But :

        Garder certains workloads sensibles ou critiques en interne (cloud privé).

        Utiliser la scalabilité du cloud public pour les pics d’activité ou services spécifiques.

    Exemple : Une banque conserve ses données sensibles dans son propre datacenter, mais utilise AWS pour héberger son site web public.
