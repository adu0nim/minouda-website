TIMEOUT - QUERY TIME OUT

Monsieur Nénert, 

Merci pour les précisions, je n'avais pas toutes ces informations dans le ticket. 
Auriez-vous des logs et des horodatages lorsque vous avez obtenu un "TimeOut" de notre API SMS s'il vous plaît ? 

Cela me permettrait de vérifier si il y a eu un incident au moment où vous avez obtenu ces erreurs.

Une remontée similaire avait été effectuée il y a plus d'un an par un autre client. Ce message "QUERY_TIME_OUT" est une erreur de client API. Ce message est renvoyé par le moteur de l’API lorsque la signature générée par le "wrapper" est trop éloignée de l’horodatage (timestamp) actuel.

Votre serveur est-il sur le bon fuseau horaire ? Avez-vous synchronisé votre serveur avec un serveur NTP pour vous assurer que vous êtes à l'heure et qu'il n'y a pas de décalage ?

Vous pouvez vérifier avec "https://time.is/" qui indiquent quelle dérive vous avez avec une source de temps fiable.

Vous pouvez aussi utiliser l'API "eu.api.ovh.com/console-old/#/auth/time" qui renvoie l’heure actuelle, du point de vue OVHcloud.

Je reste à votre disposition.
